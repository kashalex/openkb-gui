Техническое задание
Локальная система управления знаниями на базе OpenKB и GLM-4.7-Flash
1. Общая концепция проекта
1.1 Назначение системы

Разработать desktop-приложение на Python, являющееся полноценной графической оболочкой (GUI) для системы OpenKB.

Система предназначена не для классического RAG-поиска по чанкам документов, а для построения и сопровождения локальной компилируемой базы знаний (Compiled Knowledge Base / Wiki), работающей по архитектуре OpenKB.

Приложение должно:

импортировать локальные документы;
компилировать их в interlinked markdown wiki;
автоматически синтезировать concepts pages;
поддерживать summaries;
обеспечивать knowledge accumulation;
обеспечивать интеллектуальный диалог поверх wiki;
предоставлять GUI для управления всей knowledge ecosystem.
2. Архитектурная концепция
2.1 Основная философия OpenKB

OpenKB используется как:

knowledge compiler;
evolving wiki system;
knowledge graph over markdown;
long-term accumulated knowledge storage.

Система НЕ использует:

vector database;
classical embeddings retrieval;
chunk-based RAG pipeline;
top_k semantic retrieval;
hybrid search architecture.

Вместо этого система должна работать через:

compiled wiki pages;
concept synthesis;
summaries;
cross-linked markdown pages;
PageIndex;
reasoning over wiki structure.
